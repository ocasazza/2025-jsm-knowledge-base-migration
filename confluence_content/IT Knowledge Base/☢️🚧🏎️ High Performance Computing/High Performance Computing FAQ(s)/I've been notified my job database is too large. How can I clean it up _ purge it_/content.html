If your legacy jobcontrol database gets too large (disk usage, number of files or directories), it can cause high load on the submission nodes when jobcontrol processes try to scan the database, in addition to using up disk space.

You can clean your jobcontrol database with the following command. However, please see the links below to understand the implications of running this command.

$SCHRODINGER/jobcontrol -delete all

For help in cleaning up / purging your jobcontrol database, see:

  1. Legacy Jobcontrol Limitations (JOBDB)
  2. https://www.schrodinger.com/sites/default/files/s3/release/current/Documentation/html/job_control/jobcontrol_command_line_purging.htm?Highlight=clean%20jobcontrol%20database .
