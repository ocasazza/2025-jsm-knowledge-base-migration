For certain use cases you might need to add custom requests to your job submission. For jobs requiring large amount of memory to run, it's best to specify "-l mem_free=<memory_size" to avoid overloading a node as well as receiving alert messages. These are passed to the qsub command with the -l flag. For example, to request extra memory, you would need to specify -l mem_free=5G. Multiple -l options are supported for the same command invocation. Such requests can be specified in a number of places. Pick the most appropriate one for your workflow:

  • Use the -QARG option, e.g.:
  • $SCHRODINGER/jaguar run -HOST bolt -QARG "-l mem_free=5G" -PARALLEL 4 http://input.in
  • $SCHRODINGER/jaguar run -HOST bolt -QARG "-l mem_free=5G -l some_other=X" -PARALLEL 4 http://input.in
  • If you are using qsub directly, add the requests directly to your command, e.g.:
  • qsub -l mem_free=5G myscript.sh
  • If you are using qsub to submit a script, these requests can be added to #$ lines in your submission script, e.g.: bash #!/bin/bash #$ -cwd #$ -l mem\_free=5G /path/to/mycommand

true User Story

As a researcher, I need to know how to customize my job submissions with specific resource requirements so I can ensure my jobs have the resources they need to run efficiently.
