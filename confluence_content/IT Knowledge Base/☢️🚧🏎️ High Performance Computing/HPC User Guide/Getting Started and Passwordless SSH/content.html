passwordless-ssh Passwordless SSH

To ensure your jobs run smoothly on the cluster, setting up passwordless SSH within the cluster environment is essential.

If this is your first time logging into the submission node via SSH, you may be prompted to configure your SSH keys for passwordless access. When prompted, simply press return without entering any additional information. This action will enable your SSH keys to function without requiring a passphrase.

If you're unsure whether passwordless SSH is correctly configured within the cluster, please perform the following test:

  1. SSH into your machine to access the submission node.

For example, for the bolt cluster:

ssh boltsub2.schrodinger.com 2. Once logged into boltsub2, attempt to SSH back to boltsub2.schrodinger.com .

  1. If you are not prompted for a password on boltsub2, then your passwordless SSH within the cluster is correctly set up .

  2. If you are prompted for a password, you will need to set up passwordless SSH within the cluster by following Setting up Passwordless SSH from Linux and MacOS Hosts.

  3. After successfully configuring passwordless SSH within the cluster, you can copy your submission files to boltsub2 and submit your job to the cluster via the command line.

  4. If you wish to submit jobs directly to the cluster from an external machine (such as your office desktop or laptop), you must also establish passwordless SSH from that machine to the submission node. Ensure your machine is connected to the wired network in the main office or is on the VPN to access the submission node.
