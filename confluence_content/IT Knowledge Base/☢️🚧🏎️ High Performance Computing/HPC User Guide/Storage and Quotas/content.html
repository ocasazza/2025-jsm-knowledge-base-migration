Storage & Quotas

We have implemented quotas on the cluster storage. This is to help protect your ability to do your work and to encourage better data management practices. There are two areas of storage, one is only for you ( /home ), the other is shared with your group ( /nfs/working ). If you need more space, please submit a support ticket ( it-help@schrodinger.com ) with the amount of storage you need on which cluster.

Note: A large job control database can consume significant disk usage and also lead to high load on the cluster submission nodes. For information on cleaning your job control database, see this FAQ.

Checking your quota

Email alert will be sent when a user exceeds soft and hard quota usage.

bolt

The quota command doesn't work on bolt but the df command does. To see the output in human-readable format use the -h parameter.

df -h

A cron task automatically cleans up files older than 38 days from /scr on all bolt nodes. /scr is not a place to store files permanently. For storing files in long-term, please use shared NFS directories.

Home Directory

You will have a default quota of 100GB in your /home directory. This is where any files that only you care about should go. This is separate from any other home directory at Schrodinger (and each cluster is separate as well), so you'll need to scp over any files that you need ( .profile , .cshrc , data, etc). This can be expanded by requests to sysmgr.

Group Storage

You are likely assigned to a group (check with your manager or run the 'id' command if you're not sure). Each group will have a certain amount of space allotted to it in /nfs/working/group . This is to:

  • Encourage sharing of data (instead of having multiple copies of the same thing).

  • Be able to name a single person who leads the group who will justify quota increases and sort out disputes amongst group members when their space is full.

  • Avoid one group's poor storage management from negatively impacting other group's work.

Archive storage

Where you store or archive older data might depend on the standard procedures for your department or group, so check with them. If you wish to move your data to your personal or your group's archive storage space on the archive servers, see https://schrodinger.atlassian.net/wiki/x/v4zcAg for more information.

Snapshots

Periodic snapshots are taken of filesystems. You can find them in ${filesystem}/.snapshot . For example: ~/.snapshot or /nfs/working/glide/.snapshot .

File transfer to bolt cluster or from bolt cluster to other machine

  • Please use boltio node for transferring files to bolt cluster, or from bolt cluster to another machine.
