Quick Tips and FEP Tips

FEP tips

  • To save space you should delete the stage 3 directory that contains the individual fep legs IF you are sure that you have finished with the calculation and do not wish to extend it further. Please note that bolt cluster automatically deletes FEP stage 3 directories and files that are older than 30 days ( SYSMGR-46862 )

  • Generally, you will want to write the input files out from the GUI so that the correct commands are printed in the .sh script for FEP+ or the msj file for Desmond jobs:

  • fep_plus -HOST bolt_fepdriver -SUBHOST bolt_gpu -ppj -P for FEP+ jobs. The number of GPU slots should be 1, 2 or 4 only.

Quick tips

  • Store your files in your home directory in /home/ on the submission node, or /nfs/working/

  • For a large amount of data files, please only use /nfs/working/

  • ssh to the submission node

  • The first time, you may be prompted to set up your ssh keys for passwordless ssh between the submission nodes and the compute nodes. Use a blank passphrase.

  • Submission nodes: boltsub2, pdxgpusub1

  • Use the boltio node for transferring user data/files to or out of bolt cluster.

  • Builds can be found under /nfs/working/builds and /nfs/working/releases

  • Be sure to add -QARG "-P yourprojectname" to obtain higher priority

  • see the project section below for more info
