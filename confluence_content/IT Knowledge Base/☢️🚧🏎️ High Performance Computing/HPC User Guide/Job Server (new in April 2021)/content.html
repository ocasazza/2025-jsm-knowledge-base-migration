Date: Sat, Apr 3, 2021 at 12:35 PM Subject: [All] job server internal rollout - Monday April 5th To: Hi all - We've been working on a new version of job control (called job server) for a while now. With it, you no longer need to worry that bolt jobs will fail if you launch them from your laptop and disconnect from the network. Additionally, you'll find that it's more scalable — it has been a key enabler of IFD-MD development for that reason. As the next step in rolling this out to customers, we will be switching internal deployments of the 2021-1 Schrödinger Suite to use job server by default on Monday, April 5th. Launching jobs works the same as it did before, and Maestro use is very similar, but with a new monitor panel. However, the command-line interface for querying jobs is different. There are a few one-time setup steps you need to complete as a user: 1) You need to authenticate with the cluster job server (boltsub3.schrodinger.com:8030 for bolt). 2) For now, you need to run a command to start an automated download process to retrieve your results. The setup instructions depend a bit on whether you are a Maestro or command-line user and whether you are running local or cluster jobs. Please see Job Server Quick Start Instructions on Confluence for details. That page also has links to the full user docs and quick tips on the new command-line interface. If you have any questions, please reach out to us in the Job Server room on chat. We will prioritize fixing any problems people encounter, so please let us know about them.
