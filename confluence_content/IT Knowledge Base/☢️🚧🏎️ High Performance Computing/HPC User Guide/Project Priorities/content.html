project-priorities Project Priorities

The bolt cluster uses project-based scheduling. Different projects have different priorities. The queuing system will balance the usage of the cluster between the projects depending on the project's priority. Generally, this means that more of a higher priority project's jobs will start running sooner.

To assign a project priority to your job, you will need to add the following arguments to your submission commands:

-QARG "-P secret_sauce_project_x"

Where secret_sauce_project_x is the name of the project you're working on. Please contact your group leader (Lidia, Leah, Vicki, Robert, Ramy) for your specific project name and any special priority requests or changes.

If you are not associated with a project, you may submit jobs normally without the extra -QARG . Please note that the key to high priority is the association with a project.

As of 6/23/2019, the changes have been made to project configuration. Please switch to new Project Queue Name from now on.
