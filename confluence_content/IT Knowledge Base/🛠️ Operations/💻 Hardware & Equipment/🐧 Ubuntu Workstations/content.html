This page contains the recommended desktop requirements provided to IT by the DESMOND development team, broken down by use case. Depending on the workflow needs of the end user, the appropriate configuration tier should be chosen : High, Standard

Not every developer uses Linux or runs into CPU-based bottlenecks for building or running unit tests, so these are tiers that you might consider based on the developer needs of the team request. This is meant to provide the best hardware to developers as needed rather than by default to help keep hardware costs slim. Each team should be equipped with at least one developer or shared desktop for doing some development and testing on Linux. If that means a team just has a single mid-tier desktop, or many mid-tier desktops with a single high-tier desktop this is considered acceptable.

GPUs

A separate compute GPU (separate from the graphics GPU) is not required for order unless the team requesting heavily utilizes it. Teams that might want a GPU are those that work directly on or develop workflows for GPU-based back ends (like Desmond and some of the active learning workflows). This recommendation helps keep overall hardware costs low and allows us to recommend more capable CPU and RAM more broadly.

If the end-user could benefit from having another GPU in their desktop, it can always be ordered and seated later in which case see the recommendations for the Desmond GPU RTX 4000 Ada at this time for the High Tier build.

Devs need an approx. minimum of 8GB GPU memory.

Configuration Tier

Use Case

Hardware Specs

High

Primarily works on Linux and work on compiled code. End-user frequently builds the core suite and (optionally) needs a high-end GPU to develop for or run molecular dynamics jobs. Well suited for end-users that encounter clean builds multiple times a week.

Teams this might apply to:

  • DESMOND back end

  • Jaguar back end

  • Glide/Skate back end

  • PSP back end

  • Scientific Infrastructure

  • Maestro back end

CPU : Intel Xeon W5-2465X [16 physical cores/32 threads]

Compute GPU: Nvidia RTX 4000 SFF Ada

Desktop Chassis : 1350W

RAM : 4 x 16 GB

Boot Drive : 512 GB

Second Storage Drive : 2 TB (NVMe)

Graphics GPU: Existing GPU card swapped from previous workstation or A2000

High Tier Configuration - Dell Order

Standard

Primarily works on Mac or Windows, but still makes use of Linux and works on some compiled code, but mostly in python.

End-user usually builds the core suite if there's a failed NB, out-of-date swig wrapper, etc. They mostly work with a couple of unit tests and can benefit from extra parallelization when running all core suite unit tests, but don't do so all that often.

Teams this might apply to:

  • DESMOND workflow

  • Jaguar workflow

  • Glide/Skate workflow

  • Scientific Infrastructure

  • Everyone else in the Core Suite

CPU : Intel Xeon W3-2435X [8 physical cores/16 threads]

Compute/Graphics GPU: Nvidia RTX A2000 (RTX 2000 Ada not currently available yet)

Desktop Chassis : 1350W

RAM : 2 x 16 GB

Boot Drive : 512 GB

Second Storage Drive : 2 TB (NVMe)
