Some users experience issues connecting to SMB (network) shares from their local machine

Step-by-step guide

See below for steps to troubleshoot on your own. Note this only applies if you had access before and are having issues or have been recently granted access to the share. If this is your first time and still have issues your account permissions might need to be updated by a member of IT:

  1. Make sure you have an active Appgate session. See Appgate Installation for more details as needed.

    1. Sessions time out over time. Signing out/quitting the app and signing back in will refresh the session

    2. One way to check is seeing if you can access or ping internal systems like boltsub1.schrodinger.com for example

  2. Try using a different URL that connects directly to the Qumulo:

    1. PDX: smb://pdx-stor.schrodinger.com/<shared directory> instead of smb://schrodinger.com/shares/pdx/<shared directory>

      1. i.e. smb://pdx-stor.schrodinger.com/finance or smb://pdx-stor.schrodinger.com/builds

    2. NYC: smb://nyc-stor.schrodinger.com/ instead of smb://schrodinger.com/shares/nyc/<shared directory>

  3. Make sure you are trying to connect using the steps here:

    1. macOS: https://schrodinger.atlassian.net/wiki/x/lyZLAw

    2. Windows: Microsoft Support Document

  4. Perform a reboot of the local machine

  5. If all this fails please open a ticket with IT Help for someone to assist further
