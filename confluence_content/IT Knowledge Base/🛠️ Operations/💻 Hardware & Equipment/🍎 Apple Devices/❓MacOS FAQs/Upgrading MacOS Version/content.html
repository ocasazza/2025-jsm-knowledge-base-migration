32-bit Applications

Upgrading to macOS 11 (Big Sur) adds on to previous large scale updates to macOS. If you’re coming from 10.14 (Mojave) or below, please note that since 10.15 (Catalina) 32-bit applications will no longer work.

Most application developers will have already updated their apps, but you can double-check which apps you still use that may still be 32-bit by:

  1. Clicking the apple logo in the upper right hand menu bar and selecting: About This Mac

  2. Clicking the System Report... button

  3. On the left-hand side, under the Software section, click Applications

It may take a minute as macOS is gathering all the information on all installed Applications, but it will then show you your list of apps and if there are any 32-bit.

If there are any apps you still use that are 32-bit, please check the developer/company website for updates as these will not work on newer versions of macOS.

SentinelOne Anti-Virus

At Schrödinger we use the SentinelOne anti-virus program on all Macs. Upgrading to Big Sur requires a newer version of this application. After you upgrade to macOS 11 (Big Sur), a new version of the anti-virus will be pushed to your machine by Sysmgr. Immediately after upgrading, you may see your SentinelOne icon in the menu bar with an X through it:

This will be upgraded automatically by Sysmgr.

"Your Mac is up to date" or "Update not Found"

A fix has been pushed to all devices to address an issue where Big Sur has not been removed from the Software Update ignore list. If you are still seeing issues like this open Self Service(In your Applications Folder)→ Log in (Lower Left) → Utilities → unBlock Big Sur. If you do not see this option available it has already been pushed to your device. If you continue to have issues please open a SYSMGR ticket.

Terminal Crashing

After upgrading, you may experience your macOS native Terminal app crashing. If this happens, please fully quit Terminal and restart the application. If it continues to crash, please open a Sysmgr ticket.

Reinstalling macOS from bootable USB Drive

First format the USB drive to Mac OS Extended (Journaled).

  • Open Finder . Then navigate to Applications > Utilities , and open Disk Utility . You should see your flash drive under the External section in the left pane. After selecting it, click on the Erase button along the top.

  • Choose a friendly name (which you'll use later), and make sure to choose Mac OS Extended (Journaled) for the format. Finally, click Erase .

Download the macOS Installer

  • Download the desired version from the App Store or from https://support.apple.com/en-us/HT211683

Once you have a bootable usb flash drive with the desired version macOS, you can now reinstall it onto your laptop.

Boot your Macbook into Recovery mode (Turn off laptop, click power button once and hold (Command + R) buttons down until you see the Apple logo with the progress bar.

Click Disk Utilities and erase all the hard drives on that laptop and exit disk utilities. i

Click 'Reinstall macOS' from bootable drive.

  • If you do not see this option from bootable drive , then you may install it from the terminal command line. i

  • Click Utilities → Terminal

  • Run the following command & change to respective macOS version:

  • ./volumes/Install macOS <version>/Install macOS <version>.app/Contents/MacOS/InstallAssistant

Agree to the terms and conditions and select which hard drive you would like to install it on.
