If someone's Macbook has a Device Enrollment notification pop-up repeatedly, or if a Jamf installation has gone wrong, use these steps to re-enroll a computer without wiping.

Step-by-step guide

IT actions

  1. Before re-enrolling, we need to remove the current MDM profile. This can be done from the device record in Jamf Pro.
  2. Pull up the device record of the affected computer and click the Management tab.

  3. Click 'Remove MDM Profile' (This removes all configuration profiles)

    Once this is finished, this triggers the Enrollment Complete Jamf trigger, so DEPNotify will begin and the Schrodinger splash page will appear as all the initial policies begin to run again. If you don't want that to happen, just put the computer in the exception scope of the "Install DEPNotify" policy under Device Settings.


User actions

  1. To request help with the actions IT needs to perform, please see .
  2. On the client Macbook run this command in terminal: sudo profiles renew -type enrollment
  3. After running this command, a Device Enrollment notification will appear in the upper right corner or in the notification center.
  4. Tap 'Details' on the Device Enrollment notification and follow the prompts. For more details, see .
