User Never sees the Jamf Connect screen

The WiFi was not connected during laptop preference set up.

User types in their email address and Okta password and a blank screen shows up afterwards.

Check the user's Okta profile has the Jamf application assigned to them. If not, please assign that application to their account.

If that does not work, check their Active Directory account is enabled. If it is disabled, please enable their account and run a Full Import in Okta under the Directory Integrations tab for Active Directory.

Returning user can not sign in

Check the user's Okta profile if they have any previous 2Factor Authentications still registered. If so, reset them and have the user set it up again.

User forgot Okta password and still can not log in with Temporary Password

The laptop recovery key may be used here (found in Jamf). Although there are a few steps that we have to do to bypass the Jamf Login screen.

  1. Boot into Recovery Mode

  2. From the menu bar, click Utilities > Terminal.

  3. In Terminal, execute the following commands:

cd /Volumes/Macintosh\ HD/var/db/ rm ./auth.db rm ./auth.db-shm rm ./auth.db-wal

5. Restart the computer by executing the following command:
shutdown -r now

The computer restarts and the native macOS login window displays.
