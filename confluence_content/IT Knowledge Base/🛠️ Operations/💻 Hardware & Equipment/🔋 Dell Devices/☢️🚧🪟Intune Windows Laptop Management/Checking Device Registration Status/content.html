Verification Methods

Method 1: Settings Interface

  1. Press Win+I to open Settings

  2. Navigate to Accounts > Access work or school

  3. Verify the presence of a connected workplace account

  4. A connection state of "Connected to $COMPANY_DOMAIN's Azure AD" indicates successful registration

Method 2: Command Line Verification

  1. Launch an elevated Command Prompt (Run as administrator)

  2. Execute dsregcmd /status

  3. Verify the following parameters:

    • AzureAdJoined: YES

    • DomainJoined: YES/NO (depending on hybrid configuration)

    • DeviceId: [populated GUID]

    • TenantId: [populated GUID]

    • WamDefaultSet: YES

Registration Remediation

If your device shows an unregistered status:

  1. Remove any existing partial connections in Settings > Accounts > Access work or school

  2. Restart your device

  3. Follow the steps in the [Remote User Enrollment|#remote-user-enrollment] section

  4. If registration fails after multiple attempts, execute netcfg -d in an elevated Command Prompt, restart, and retry
