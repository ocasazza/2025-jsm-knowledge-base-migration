Please note, this page is meant to share information between IT and for the management of Visual Studio licenses, and not intended as instructions for end users of Visual Studio licenses

INLINE

We license Visual Studio Professional for a subset of our developers, with usage being tracked via AD group membership and renewed based on usage.

For End Users

Core Infrastructure > Build Machines > Windows build machines

Sysmgr Instructions

Approvals: See Visual Studio licenses

Standalone Licenses

  1. In Active Directory, add the user to the group "App_Visual_Studio_Licensees"

  2. If the user will only be spinning up Windows build machines in Spinner, you're done. Otherwise:

    1. Provide the user with one of the license keys in Software License Keys, sheet two: Microsoft .

      1. Hint: Search for "Visual Studio Professional 2019"

      2. Use any of the three keys as we only track the qty of users, but note which key they're using .

    2. Provide the user a link to download the software: https://visualstudio.microsoft.com/vs/

The old way of managing licenses is documented here: Visual Studio fulfillment/assignment instructions

Subscription Based Licenses

  1. Sign into Microsoft's Visual Studio Subscriptions Admin Portal - Manage Subscribers . Use the it@schrodinger.com account listed under "Microsoft VLSC/Azure" in LastPass and read the LastPass notes on how to login.

  2. Image for the following instructions:

  3. Make sure the correct organization is selected in the drop-down next to the username.  As of , we have Schrodinger, Inc, and Schrodinger LiveDesign.  For Visual Studio subsciptions, you should normally select Schrodinger, Inc. from the drop-down.

  4. In the drop-down next to Add agreement, ensure the correct agreement is selected.

    1. OLV: V8158887 from SYSMGR-61196 b2a2abc6-677a-31ef-bef3-17f162f52bcb System Jira

    2. OLV: V6521913 from SYSMGR-62859 b2a2abc6-677a-31ef-bef3-17f162f52bcb System Jira

  5. Under All subscriptions, click +Add and provide the username and email address of the user, or bulk add.
    Note: The sign-in address will need to match what's set for them in Azure Active Directory. Generally first.last@schrodinger.com but it could be username@schrodinger.com .

  6. Under " Subscriber can access software downloads and product keys", make sure that's set to Allow .
