Activating your Druva inSync client Linux should take less than two minutes, as you'll just need to launch the inSync client and log into it with your Okta credentials.

Please find the detailed instructions with screenshots below.

Please note, any directories named 'donotbackup' will not be backed up.

If you do wish to move files into 'donotbackup' directories, do this before proceeding below.

Confirm that the software is installed

  • Unless you have a self-managed desktop, you will not need to install the inSync client manually on your laptop or desktop.

  • The IT department will have alreedy installed the client software via an automated deployment.

  • To confirm the software is installed, you should see the following:

    • An inSync client System Tray icon in the lower right corner (assuming you’re using the default KDE desktop).

    • Notice a red exclamation point on the inSync Icon? Thais indicates that the inSync client is install but not-yet activated.

Before Attempting to Activate the inSync client

Please first check these two items, otherwise, you might have issues after clicking “Activate” on the inSync client.

  1. Please make sure you have either Firefox (/usr/bin/firefox) or Google Chrome/Chromium (/opt/google/chrome/chrome or /usr/bin/chromium-browser) as your default web browser for the inSync client to authenticate correctly. Figures a, b, and c below should help you find how to set your desktop's default web browser:

  2. After you've set your Linux desktop's default web browser to either Firefox, Chrome, or Chromium, please open it and sign into your schrodinger.okta.com account.

Once you're logged into your preferred browser, please proceed to schrodinger.okta.com and log into your Okta account.

Activation

Once the inSync client software is installed on your computer, it needs to be activated by the owner of the system. In general, this involves opening inSync client, entering your first.last@schrodinger.com email address, and signing into Okta in your browser when prompted (works best if you're already signed into Okta). Some specific instructions are included below (click the expand for details).

Once you activate the software, the backups will start.

You may notice the first backup will have a more noticeable impact on your computer than subsequent ones.

  1. Launch the inSync client by clicking on the Druva inSync icon in your System Tray.

  2. Enter your first.last@schrodinger.com email address, but leave the password field blank. Then click “Activate” to continue.

  3. Because you already have the system’s default web browser open with an open tab which is already signed into your schrodinger.okta.com account (from the "Before Attempting Activate the inSync client" step above), the authentication should complete and display an “open with” dialog

    2. Soon after you see the successfully activated Druva webpage, you should get a pop-up asking how you would like to open the redirected link. Please choose to open the activation link as follows:

      The “Open link” window from Firefox The “Open xdg-open” window from Chrome


  4. After the authentication completes you will get redirected back to the inSync client.

    1. If you don’t get forward back to the inSync client, please ensure that you are already signed into your Schrodinger OKta account and that you also have the OKta plugin installed for that browser

    2. If you are still having trouble with the browser sending the authentication back to the inSync client, please contact IT in the Druva inSync help (alpha and beta periods) google chat room.

  5. Once you’ve been signed into the inSync client, if you do not already have another device enrolled, the backup will start.

    1. If you do already have another device enrolled, choose "Add as New Device" and click “Continue”.

  6. With the activation process completed, and the new device now added, the inSync client will start initializing any /scr* backups you may have on your Linux desktop.

  7. Also notice that once you’ve activated the inSync client, the System Tray icon no longer has the red exclamation point.

    1. It will also flash if it’s syncing data.

  8. Please note that the inSync client will only work under a GUI session .

    1. If you’re logged into the desktop’s GUI environment IRL, it will be working in the background.

    2. If you’re logged into the desktop'S GUI environment remotely (X2Go/FreeNX/NoMachine/VNC), it will be working in the background.

    3. If you're logged into the desktop's GUI environment remotely, and you close the remote window but never logout , it will be working in the background.

    4. If you’re logged into the desktop via ssh only, it will NOT be working in the background!!!
