Overview

Druva inSync is a cloud-based backup solution that Schrödinger has deployed to create remote backups of users' files on their local machines and stored in Google Drive, and give users the ability to restore their own files on-demand.

The client is installed and configured remotely to be as un-intrusive as possible while giving users the ability to configure some options, pause backups, and include or exclude files beyond the default.

Web Interface

From the web interface you can download or restore files and folders from any of your backups.

This can be particularly helpful if you're trying to retrieve files from a system other than the one your client is running on, or if your primary system is unavailable.

Logging in

The web interface for your Druva account can be accessed via Okta or by logging in directly at https://cloud.druva.com/home/ :

  1. Go to https://cloud.druva.com/home/

  2. For username, enter the same Schrödinger email address you use to sign into https://schrodinger.okta.com/ . Leave the password field blank , then click Login.

    1. If you were already signed into Okta in the same browser, you'll be logged into the inSync web interface.

    2. If you weren't already signed into Okta, you'll be prompted to sign into Okta, and then you'll be logged into the inSync web interface

InSync Client Activation

The inSync client must be installed and activated on devices (laptops, desktops) which are to be backed up.

Launching and Using the inSync Client

In your taskbar or menubar you'll see an icon resembling a pinwheel or a counter-clockwise arrow (depending on your platform).
Click on that and select "Open Preferences…" to bring up the client interface.

Backup Settings

After launching the inSync client, you can click on Settings to see the backup settings.

For example (Windows client, version 7.1):

In addition to those settings, please note:

  1. The operating system and applications are not backed up by inSync.

  2. Your home folder on Mac and Windows, and only /scr* (not /home/ username ) on Linux will be backed up, with some exclusions.  Those exclusions include:

    1. Anything placed in a folder named donotbackup anywhere in the location being backed up will be excluded. You can have more than one folder named donotbackup .

    2. Paths listed in the Global exclusions, which can be found using these instructions: How to View global exclusions

    3. Mac: System Applications will not be backed up

Adding backup locations

You can also add folders to be backed up, if they are not already included in the locations above. See How to Add folders .

Retention times

These cannot be adjusted by the user. Note : The last snapshot is saved indefinitely irrespective of the retention policy configured below.  That last snapshot is only removed if the device or account is deleted from inSync.

Backing Up

The default backup settings will schedule backups to run periodically and automatically. If you need to run a backup manually, launch the inSync client, click on Backup & Restore, and click Backup.
