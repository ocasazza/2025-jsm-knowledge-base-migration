This document explains how to reconnect your device (whether it be a Windows or Mac device) back to our Automox Environment. Automox is a platform that automates the management of endpoints, including patching, configuration, and security.

For Windows

Open the Task Manager, navigate to the "Services" tab, and look for the "Automox Agent" service; if it's listed as "Running," the agent is active; if not, you can right-click and select "Start" to initiate it.

For safe measure, please restart your device here as well.

For Linux/Ubuntu

When Devices still show a disconnection:

When devices are still disconnected after restarting the agent and rebooting your device, there may be an incompatibility between the device itself and the Automox environment. Following the compatibility checklist is a prerequisite to establishing proper connection between two.

Compatibility Checklist (Agent Requirements):

For Mac:

-3GB of Disk Space Free

-Mac device requires Secure Token User Permission

-Connectivity to Patch source on Device: Mac App Store

For Windows:

-3GB of Disk Space Free

-Powershell Installed

-WMI Check

-Connectivity to Patch source on Device: Windows Update Server
