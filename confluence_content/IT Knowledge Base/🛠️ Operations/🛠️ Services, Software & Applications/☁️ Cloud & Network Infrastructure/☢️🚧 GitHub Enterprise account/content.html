How to get access to Schrodinger Enterprise account?

User access to the S chrodinger org is managed by Okta.

  • Open a Service Request in Freshservice to get access to GitHub Enterprise Okta app

  • Once the app is assigned to your Okta dashboard, click on the app.

Initial Setup

  • For GitHub, you will need to have a GitHub account.

    • You can either use an existing personal account that you have, or create one specific for your work at Schrodinger .

    • Probably the main thing to note is that once you start contributing to Schrodinger GitHub repos, you will get a lot of automated emails when people comment on repos you're active on. These emails are valuable, and it's best not to silence them. So:

      • The advantage of using a work-only GitHub account is that these emails will be sent automatically to your work email. However you will have to juggle two GitHub accounts.

      • If you use a personal account, you can redirect all Schrodinger related GitHub emails to your work email. Here's how. In your GitHub settings, under Emails, add your work email as an additional email address. Then, after you've been added to the Schrodinger organization on GitHub, so to Notifications in settings, and go all the way down to Custom routing. Put your work email as the email address for the Schrodinger organization.

  • Add your local SSH key to this GitHub account

https://docs.github.com/en/authentication/connecting-to-github-with-ssh/adding-a-new-ssh-key-to-your-github-account

  • Configure your local SSH key to be authorized for SSO

https://docs.github.com/en/enterprise-cloud@latest/authentication/authenticating-with-saml-single-sign-on/authorizing-an-ssh-key-for-use-with-saml-single-sign-on

GitHub Copilot

  • Please see Configuring Github Copilot for instructions for requesting a GitHub Copilot license

Warning

GitHub Enterprise SSO linked with Okta but it does not manage your Github credentials and Two Factor authentication.

User credentials for individual accounts are still managed by Github and IT does not have any access to administer your accounts. Please follow the steps listed on GitHub Support to start the recovery process.

Note to remember your Github password, any recovery keys and do not remove your multi-factor authentication from Github.

    • There will be required to log into your account if you logout or need to re-authenticate.
