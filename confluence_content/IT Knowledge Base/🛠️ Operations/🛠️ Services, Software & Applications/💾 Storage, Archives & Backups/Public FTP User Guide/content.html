Posting a file for an external user to download

If you have a file that is too large to send via email, you can place it on the ftp site to allow an external user to download it. In NYC, place the file in:

/]]>

where <somedirectory> should be a descriptive name (the institution of the person that will be downloading, for example).

You can access /nfs/ftp from any NYC Linux machine; some example machines are listed on the IT User Orientation page.

Give world execute permissions on <somedirectory> , but remove world read permissions (we do this to prevent people from listing/browsing the contents of the directory). You must give world read permissions on the file itself, however.

You'll then need to send the user the full URL path including the filename. They will not be able to list or browse the directory contents via command-line ftp or their GUI ftp client/web browser.

If someone needs to send us a file that is too large to send via email, they can upload the file to our ftp site. The user should upload the file to the incoming directory of ftp.schrodinger.com .
