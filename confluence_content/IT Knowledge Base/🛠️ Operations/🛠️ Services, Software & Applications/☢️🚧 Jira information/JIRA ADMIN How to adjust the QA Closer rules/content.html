How to adjust the rules which set the QA Closer field in Jira

Step-by-step guide

Add the steps involved:

  1. Several Jira Automation rules control the functionality of automatically setting the QA Closer field for new tickets.
  2. There is a "general rule" owned by IT: https://schrodinger.atlassian.net/secure/AutomationGlobalAdminAction!default.jspa#/rule/79 . Prior iterations were created in System JIRA ac6def98-2140-30c3-8103-4b52b8b31135 ITDW-1150 .
  3. For certain products with special considerations (usually, where the QA Closer is not solely based on Product, but also by Component), the QA Closer is set by product-specific rules owned by QA:
    1. PANEL - https://schrodinger.atlassian.net/secure/AutomationGlobalAdminAction!default.jspa#/rule/20
    2. PRIME - https://schrodinger.atlassian.net/secure/AutomationGlobalAdminAction!default.jspa#/rule/77
