The steps up to the end of the activation section should take less than two minutes, as you'll just need to launch the inSync client and log into it via Okta.

Confirm software is installed

You should normally not need to install the inSync client manually on your laptop or desktop. IT will have installed the client software via our automated deployment systems.  If you do not see the software already installed, you may need to manually reboot to help the installation complete. The installation will not reboot your computer automatically.

Before Activating

Please note, any directories named 'donotbackup' will not be backed up. If there is content in your home directory on your Windows machine you do not wish to be backed up, please move it into a directory named 'donotbackup' before activating and starting your backups.

The backup and restore processes will not prevent your machine from sleeping. See the platform-specific instructions for tips on how to prevent your machine from sleeping.

Activation

Once you activate the software as described below, the backups will start.  You may notice the first backup will have a more noticeable impact on your computer than subsequent ones.

Once the inSync client software is installed on your computer, it needs to be activated. In general, this involves signing into the client by entering your first.last@schrodinger.com email address, and signing into Okta in your browser when prompted. Some specific instructions are included below.

  1. Launch the inSync client from the Druva inSync icon on your desktop. Enter your first.last@schrodinger.com email address and click Continue.

  2. Your default web browser will open, and you'll be prompted to sign into Okta, if you're not already signed into Okta in that browser:

  3. Once you've signed into Okta, you should see a browser prompt asking if you'd like to let the cloud.druva.com website open the inSyncAgent.exe program on your computer.  You should Allow this.

  4. The Druva inSync program will open.

  5. If you do not already have another device enrolled, the backup will just start.

  6. If you do already have another device enrolled, you'll be given a choice.

    1. If you're enrolling another device, rather than replacing an existing one, choose "Add as New Device". The backup will start:

    2. If you're replacing an old device with the new one, choose Replace an Existing Device .   The backups associated with the old device will be switched over to be associated with the new device. Please note, if you choose to replace, backups will stop on the old device. You can see more instructions here .  Please contact sysmgr if you have any questions.

  7. Go to Druva inSync Cloud Overview for other instructions.

If you plan on leaving your device to restore data unattended then please set your computer to not fall asleep when connected to power. Restores pause when a computer fall asleep.
