We have the option to use GCP labels and AWS tags to define how resources are accessible via Appgate. Users can add and remove these tags at will in order to control access to the resource from Appgate.

Tags are based off of our Appgate gateway locations ; you may need to allow respective IPs from the gateways in order to access your resource properly if your resource uses IP safelisting.

How to create a tag

Reference the below tables to create a tag for your instance or application. If you're unsure of what tag you should use, reach out to InfoSec in the Security Hotline Google Chat channel. We generate tags using the below template; tags are case-sensitive.

Each resource will need at least 2 tags in order to function. One tag will indicate where to route the request and another will indicate what groups of people are allowed to access which protocols groups.

Tag

Value

appgate-site

Site Code

appgate-access

Protocol Group and Department or Department Group separated by a dash "-"

Site Locations

In order to optimize speed and accessibility, choose a site that corresponds most closely with the instance or application you're trying to control access to. Please note that if your resource is in AWS, it must route through an AWS hosted site. Similarly if your resources are in GCP, they must route through a GCP site.

We group protocols by general function for the sake of simplicity. We currently support the below protocols groups but can work with you if you need additional ports or groups added. We encourage managing your own access and adding/revoking access as needed.

Examples

Scenario #1: You have a web application hosted in GCP US-East where you'd like to expose the web UI publicly but only want it to be accessible from within Schrodinger. In this case, you'd add the below tag which will route all traffic publicly through Appgate.

Tag(s)

Description

appgate-site:guse
appgate-access:web-ei
appgate-access:admin-dev

Allows HTTPS access for the Enterprise Informatics department routed publicly through the IT-managed GCP region US-East.
Allow SSH/RDP access to the "Dev" group.
