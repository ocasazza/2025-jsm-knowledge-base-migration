Appgate Gateways Table

If you are going to safe-list Appgate Gateways for your service or application, first pick the region that is "closest" to your service. Then find the public IP addresses for the appropriate pair of Gateways (please prefer those in GCP or AWS!) and add those to your firewall rules or security groups. Once an entitlement is created by the Infosec team, your application will be reachable via Appgate. Please note the "External IP" is the egress IP of the gateway in question, and may not be what the DNS name resolves to.

It is recommended for AWS Security Groups or GCP Firewall rules you safelist all IP addresses listed under the External IP column for your resource. This will ensure consistent connectivity via Appgate. For easy of copy-pasting, a dynamic list of all such IP addresses can be found .
