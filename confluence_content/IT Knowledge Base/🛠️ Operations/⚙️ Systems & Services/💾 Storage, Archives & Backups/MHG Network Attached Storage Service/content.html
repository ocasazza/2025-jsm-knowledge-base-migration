The purpose of this documentation is to provide a general overview of MHG Network Attached Storage service. Quota is applied on all shared folder, more information about the latest quota settings can be found in the table below.

Available Service(s):

  • data sharing via NFS
    • shared resources:
      • users' home directory
      • Schrodinger Suites (OB/NB/Releases)
      • Databases
      • Schrodinger tools and utils (sysmgr,utils)

Operations

New Project folder request

It is highly recommended that a customer service project has a dedicated volume.

To request a new project folder requires:

  1. a SYSMGR ticket in which contains:
    1. project name
    2. project owner
    3. list of users who are working on the project
    4. volume size in GB *

It might take a few days to complete this request.

* : approval might be required (see )

Change quota setting of a shared folder

This request is available for all shares mounted in /nfs/working folder! The Home folders' quota is predefined, reviewed regularly and it cannot be changed by a simple SYSMGR request.

  1. Open a SYSMGR ticket with the following subject: "CM_Request: MHG-QUOTA_<PROJECT_NAME>" (replace "<PROJECT_NAME>" with the name of the project/folder in /nfs/working)
    1. In the ticket body add: the new quota in GB
    2. If manager's approval required, please add him/her to the ticket as watcher or mention him/her on the ticket.

Archive a project

It is possible to archive projects in MHG if there's a legal requirement to keep the data local . This process is automated if the project has its own dedicated volume and it can be initiated by creating a file called ".archive_ready" in the root of the project folder. E.g. /nfs/working/project_1/.archive_ready

The content of .archive_ready file: the retention period given in years. E.g. 5 years = 5y

If the project doesn't have its own dedicated volume (e.g. it is located in /nfs/working/appsci): a SYSMGR ticket has to be opened.

Restore an archived project

Restoring an archive project requires:

  1. a SYSMGR ticket in which defined:
    1. project name
    2. archive date
  2. a manager approval (head of department)
  3. a business case

To restore an archived project might take one week up to a month.
