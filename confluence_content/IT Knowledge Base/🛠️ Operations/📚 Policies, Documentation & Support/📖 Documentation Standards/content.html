Organization & Templates

Each page should be well scoped to one item or topic. This is a deliberate choice that will ensure our IT Service Desk portal will suggest individual pages that do not contain pieces of information outside the scope of their request.

Each page should contain the following items:

  • Tags that match the contents of the page. Ensure any “kb-*” tags are correctly assigned ( see Automations below ).

  • Excerpt macro at the beginning of the page that contains a summary. You can use the AI summary feature to quickly generate this.

Tone & Voice

  • Please try and use concise, formal language.

    • Avoid using personal pronouns.

    • Use factual statements.

    • Do not add unnecessary language.

  • Be DRY (don’t repeat yourself)

  • If we’re asking people to read a document, we want to make the transfer of information as quick and simple as possible, while still communicating the correct information.

Formatting

  • Please avoid unnecessary line breaks. The user needing to scroll is negative experience. In an ideal world, an entire page will fit on screen.

  • Please use Title Case for page titles.

Automations

The following automations have been created to help IT Operations members create documentation that follows our documentation standards and guidelines.

Smart Labels

The following labels are used in the knowledge base to identify documents. The Knowledge Base Smart Label Automation can be applied manually by right clicking on page options in the left hand toolbar and selecting the automation. In addition the automation is triggered when pages in the ITLC space are published.

Tag

Description

kb-configuration

This page specifically relates to the configuration or modification of resource, application, or other system in order to change the function of that system to meet a user's needs.

kb-how-to-article

This page specifically provides steps or procedures or other instructions in order to accomplish a specific action.

kb-project

Does this document generally relate to the IT operations knowledge base and/or Is this page part of the IT Operations knowledge base?

kb-troubleshooting-article

This page is a knowledge base article that helps users identify and resolve specific problems they may encounter with a product or service. This articles provides detailed diagnostic guidance to assist users in pinpointing the root cause of an issue, followed by clear, step-by-step solutions.

kb-architecture

This page is a conceptual model that defines the structure, behavior, and views of a system and/or this page is a formal description and representation of a system, organized in a way that supports reasoning about the structures and behaviors of the system.

kb-automation

Does this page describe a process of creating software or systems to replace repeatable processes and reduce manual intervention for users?

kb-integration

This page describes bringing together software sub-systems, to create a unified single system for data analysis, reporting, and other business operations.

kb-lab-device

A tool or instrument used in scientific research, such as biology, medical science, chemistry, and physics, to perform various measurements and experiments

kb-meeting-notes

A written record of what was discussed and decided during a meeting, typically including the date and time of the meeting, a list of attendees, a summary of the topics discussed, decisions made, action items assigned, and the time of adjournment.

kb-policy

This document outlines rules and procedures for using organizational technology systems, including acceptable use of computers and data, security requirements, and consequences for violations. It contains formal directives about user responsibilities, access controls, data privacy, and incident reporting procedures.

kb-hardware

This page provides detailed specifications and usage information for company-approved computing devices and accessories, including laptops, workstations, monitors, and peripherals. It contains technical details, model numbers, and standard configurations for supported hardware that employees may use or request.
