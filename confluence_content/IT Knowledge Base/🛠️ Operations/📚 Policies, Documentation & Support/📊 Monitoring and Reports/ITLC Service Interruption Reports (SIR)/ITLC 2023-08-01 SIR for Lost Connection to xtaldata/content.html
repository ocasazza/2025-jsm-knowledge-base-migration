Executive summary

xtaldata is a legacy linux system that mounts NFS shares from the legacy NMA Synology devices, and also export NFS shares to a few other legacy desktop in NMA use.

Early 08/01 xtaldata wasn't reachable after getting rebooted.

Users reported that they couldn't reach xtaldata and the some shares were not usable .

After troubleshooting we found that the network wall port wasn't passing through a stable connection to the network switch, so we moved the ethernet cable from xtaldata to another wall port and that made the system usable again.

Background

xtaldata is a legacy linux system that will be replaced by nma-xtaldata-l01 once we move from NMA to HAM

Chronology

10:27AM EDT - xtaldata was rebooted

4:26PM EDT - reported to that she wasn't able to reach xtaldata (info taken from the ITINFRA-1826 ticket only)

4:30PM EDT - I posted in ITINFRA-1826 that i wasn't able to to ping xtaldata but I was able to ping other devices on the legacy VLAN (192.168.200.0/24)

6:05PM EDT - After chatting with it was determined that the issue was hardware related, but wasn't sure if it was an issue with the NIC (on either end), the network Ethernet cable, or something about the wall port's patch work.

11:30AM EDT - was able to get xtaldata back on the network by using the same Ethernet cable on a different network wall port.


Corrective Action

Moved xtaldata's network cable to another wall port

Analysis

Some wall ports in the old NMA office aren't well patched or have come loose over time.
