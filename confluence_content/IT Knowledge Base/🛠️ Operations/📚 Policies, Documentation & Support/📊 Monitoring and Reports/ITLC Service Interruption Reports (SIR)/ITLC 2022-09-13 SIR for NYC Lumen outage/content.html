Executive summary

  • NYC primary link(Lumen) got failed due to the Linecard issue on the ISP end.
  • Raised a ISP ticket ( 24886349) with the LUMEN.
  • Failover was performed to the secondary ISP (GTT)  to make  connectivity to restore back in the NYC.

Background

  • Lumen is the Primary ISP in the NYC which went down due to the ISP issues.
  • Took call to failover to the secondary circuit (GTT).
  • Once the site is on the secondary circuit Wifi got restored.
  • Changes are made on the DNS part to the secondary Egress ip  in NSone to restore the Appgate connectivity.
  • Communicated the Users to make the secondary ISP Egress ip to get whitelisted for the Internet resources.


Incident Description

PrimaryISP (Lumen)  is down due to the ISP issue.

Failover to the secondary isp :- @10:45 AM EST

Appgate failover to the secondary isp :- @11:00 AM EST

Issue fixed and from the ISP and Failover to the primary ISP :- @22:00 PM EST


Corrective Action

  • As per the current design manual intervention is required for the failover of the site .Automatic failover is possible if we have the BGP peering with the ISP.
  • During the issue, found issues to access the firewall using the OOB Network  Need to get this documented( During issue i was able to access using the secondary tunnel from the Hyderabad  firewall)
  • Few sites lost connectivity to the NYC since they have only one tunnel connectivity . Need to revisit those Branch site and build the redundant tunnels to the NYC have the connectivity if failover is done.
  • Need  some documentation on the APPGATE connectivity of the NYC site.
  • Need to educated the end users to whitelist  the secondary  egress ip (GTT) to access the internet accessible resources during the failover.
