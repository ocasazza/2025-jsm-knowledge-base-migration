Executive summary

User reported Confluence not display contents as expected, but instead of following error:


IT manage quickly checked and restarting Confluence service. After restarting Confluence, Confluence service returns to normal.

Background

This outage was triggered by replacing some plugins, but not restarting Confluence immediately.  The plugins were replaced manually while working on INFOSEC-1343.

Incident Description

The incident occurred soon after some of the plugin packages were updated.

Chronology

3:47PM PDT User reported Confluence issue in IT hotline chat room

3:50PM PDT IT manager responded and started investigation

4:10PM PDT Confluence issue is resolved. Confluence service is restored. Users confirmed it works again for them

Corrective Action

IT manager checked Confluence log and then restarted Confluence service. Restarting Confluence service fixed the issue

Analysis

IT manager looked at Confluence, but did not find anything standout.  The root cause was replacing commons-text -1.8.jar with commons-text-1.10.jar while Confluence was running.
