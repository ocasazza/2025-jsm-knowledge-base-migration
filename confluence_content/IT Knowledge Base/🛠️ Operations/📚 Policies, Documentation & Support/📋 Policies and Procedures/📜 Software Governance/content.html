All software installations and purchases must be reviewed and approved by the IT Department and Information Security. It is policy to raise all software requests with IT prior to moving forward with steps such as final vendor selection, security review, legal review, signatures, or installations.

Please note: This process is specific to software being brought into the organization for employee use, and does not replace the process to bring in software for inclusion with Schrödinger products.

Fast Track

If the Software Request meets one of the following criteria, you may skip ahead to Step 3 - Submit Information to the PM Team

  1. Is a Large Software Application (New or Replacement)
  2. Has a fee/cost associated
  3. Requires a new SOW (Statement of Work) and/or contract
  4. Requires integrations with other applications, either internal or external

If you are unsure, please start at Step 1 - Submit an IT Request .


Purpose

If your team has a business need for a new software application - please follow the process below.  This process will ensure all software applications introduced into the Schrödinger ecosystem are properly vetted and approved.

Depending on the request, it may require IT resources.  Advanced notice will allow the IT Department to prioritize your project within our portfolio and plan work based on resource availability.

Process

  1. Submit an IT Request

    1. Navigate to IT Help and select Service Requests .
    2. Within the Service Catalog, select Software Requests from the left-hand menu and search the applications for your current request.
      1. If you found the application you're looking for, submit a request for the software existing in the catalog.
      2. If still not found, submit a request via the + Other Software Requests tile.

  2. IT Ops Review

    1. If the request is within the scope of the IT Service Catalog, the standard process is followed. You'll hear from an agent who will work on the ticket to completion.
    2. If the request is deemed out of scope for the IT Ops Team to complete, you will be advised to submit additional information to the Project Management Team (PM Team), detailed in Step 3.

  3. Submit Information to the PM Team

    1. Submit the Asana form , along with a completed Project Scoping Document , as supporting material.
    2. Please reach out to the project-management-group@schrodinger.com with any questions.

  4. PM Team Review

    1. A PM will review the request and ask for additional information, as needed.
    2. The PM will facilitate preliminary analysis, including, but not limited to:
      1. Understanding the analysis that has already been performed by the requestor
      2. Confirming there is EVP or SVP sponsorship for this request
      3. Confirming the request is included in the AOP
      4. Investigating if Schrödinger already has a similar tool that can be leveraged
      5. Brainstorming alternate solutions
    3. The PM will help the requestor compile sufficient information to present to the Project Planning Team (PPT).
      1. The Project Planning Team meets bi-weekly and is made up of all IT Managers, as well as all of the IT Project Managers.

  5. PPT Review and Prioritization

    1. As the Requestor, you'll be asked to share project details with the PPT.
    2. The PPT will raise questions with a goal of understanding the business need and priority.
    3. If the project is approved, p lanning will occur to identify when the project will be able to start, taking into account the teams capacity and project priority.
    4. A PM will contact the requestor with next steps. Depending on the size of the project, all or a subset of the next steps will be managed by a PM.

  6. Software/Vendor Selection

    1. If an NDA does not already exist with the proposed vendor(s), the Project Manager will initiate the NDA process.
    2. The level of analysis will vary based on the size of the project. For example, large projects may have to go through an RFP process.
    3. Once a short list of one or two vendors has been finalized, the required reviews below can proceed.

  7. Technical Review & Security Review

    1. The Digital Workspace (DW) team will perform a Technical Review to understand technical capabilities of the proposed software.
    2. Then the InfoSec Team will perform a Security Review, which is initiated by the InfoSec Vendor Assessment Request form .

  8. Legal Review

    1. Once down to a single recommended vendor, the Legal team will be engaged to review the contracts.
    2. Legal will work with the counterparty until all terms are agreed upon.

  9. Route for Signature

    1. An implementation approach and timing of the project will be confirmed prior to signing the contract.
    2. The PM will send the Signature Request to Legal Operations to formally route the contract for signature. This allows for confirmation that the approved version of the contract is being signed.
      1. At this time, only Shane Brauner and Pat Lorton are authorized to sign software purchase contracts, regardless of contract dollar amount.
      2. If there is an existing signature delegation that applies to this purchase, it will be reviewed by Legal Ops prior to routing for signature.
    3. Once the contract is signed by all parties, Legal Ops will file the executed contract in Box.

  10. Add new Vendor to Coupa

    1. The PM will ensure the Coupa Process is initiated, to ensure payments can be processed.
