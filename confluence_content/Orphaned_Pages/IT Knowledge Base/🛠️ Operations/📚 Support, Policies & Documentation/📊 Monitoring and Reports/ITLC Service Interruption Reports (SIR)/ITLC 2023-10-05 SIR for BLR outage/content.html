Executive summary

Received the pager duty alert for the BLR site showing down. Issue  is with the ACT which is the primary ISP in  BLR had the fibre cut during the outage.

Background

  1. ISP had the fibre cut in the location which impacted the services  on the primary isp.
  2. Site is configured with the redundancy , so the site is up via secondary isp.
  3. Issue with the routing for the monitoring with the secondary ISP, which is corrected during the issue and the monitoring is up via secondary also.
  4. BLR Site has the PPOE connections so no manual intervention is required for the failover.

Incident Description

Act is the primary ISP which is having the fibre cut in the location caused the monitoring down for the site.

Chronology

Sep-06-2023


  • Received the pagerduty alert from the Librenms and informed in the IT-Core chat channel
  • looked after that and informed the site is up on the secondary link and updated issue with the Monitoring
  • Looked after the routing and fixed the issue with the routing on the secondary link to fix the monitoring issue


Corrective Action

Site is having the redundancy,  issue with the monitoring over the secondary link and it is also fixed during the  time of the issue.

Analysis

Found the Secondary link is having the Asymmetric routing to wards the librenms and corrected this
